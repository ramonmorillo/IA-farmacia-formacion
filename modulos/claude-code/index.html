M5 - Claude Code para Farmacéuticos

Cómo pedir cambios en esta plataforma sin saber programar

🎯 Objetivos de aprendizaje

  • Entender qué es Claude Code y para qué sirve en este contexto
  • Saber formular peticiones claras que produzcan buenos resultados
  • Reconocer y evitar los errores más frecuentes al pedir cambios
  • Seguir un ejemplo práctico completo de principio a fin

Tiempo estimado: 20 minutos Sin conocimientos técnicos requeridos

¿Qué es Claude Code?

Claude Code es una herramienta de Anthropic (la empresa que fabrica el modelo de IA Claude) que permite modificar proyectos de software directamente mediante instrucciones en lenguaje natural. No necesitas saber programación: describes lo que quieres y Claude Code hace los cambios.

💡

Analogía para farmacéuticos:

Imagina que tienes un técnico informático disponible 24 horas, que habla español perfectamente, que conoce todos los archivos de tu plataforma y que ejecuta exactamente lo que le pides. Claude Code es ese técnico, con una diferencia importante: tú debes revisar su trabajo antes de darlo por bueno, igual que revisarías una dispensación antes de entregarla.

¿Cómo funciona en la práctica?

Claude Code opera directamente sobre los ficheros del proyecto (los archivos HTML, CSS y otros que forman esta web). Cuando le pides un cambio:

  1. Lee los archivos relevantes para entender el contexto actual
  2. Planifica qué modificar y en qué orden
  3. Hace los cambios en los archivos directamente
  4. Te explica qué ha hecho y por qué
  5. Sube los cambios al repositorio del proyecto
⚠️

Importante: Claude Code actúa, no solo responde. A diferencia de un chatbot donde la conversación no tiene efectos fuera de la pantalla, aquí los cambios se aplican a archivos reales. Por eso es fundamental revisar el resultado antes de aceptarlo.

Para qué sirve en esta plataforma

Esta plataforma formativa está hecha con archivos HTML y CSS. No tiene base de datos ni panel de administración. Para modificarla hay que tocar los archivos directamente, algo que normalmente requiere un perfil técnico. Claude Code elimina esa barrera.

Usos habituales que puedes pedirle

✓

Contenido y texto

  • Corregir erratas o actualizar información en cualquier módulo
  • Añadir un nuevo ejemplo o caso práctico a un módulo existente
  • Cambiar la redacción de una sección para que sea más clara
  • Actualizar fechas, referencias o nombres de medicamentos
✓

Estructura y nuevas secciones

  • Crear un módulo formativo nuevo completo
  • Añadir una sección dentro de un módulo ya existente
  • Crear un nuevo caso interactivo con sus opciones y retroalimentación
  • Añadir un perfil nuevo (por ejemplo, "Farmacia Rural")
✓

Navegación e integración

  • Añadir un módulo nuevo al menú de la portada
  • Enlazar páginas entre sí
  • Actualizar el footer o el aviso legal
  • Modificar la versión y fecha visibles en la portada
✗

Lo que Claude Code no hace (ni debe hacer) por ti

  • Verificar la exactitud científica del contenido — si añades información farmacológica, eres tú quien debe revisarla con fuentes validadas
  • Tomar decisiones editoriales — qué incluir o no es una decisión profesional tuya
  • Publicar cambios sin tu aprobación — siempre hay una fase de revisión antes de que algo llegue a los usuarios

Cómo pedir cambios correctamente

La calidad del resultado depende directamente de cómo describes lo que quieres. No hace falta vocabulario técnico, pero sí claridad y contexto.

Los cuatro ingredientes de una buena petición

1

Dónde: localiza el lugar

Especifica en qué módulo, sección o página quieres el cambio.

Vago: "Cambia el ejemplo de interacciones"

Concreto: "En el módulo M2 (Casos de Uso), en la sección de usos permitidos, cambia el ejemplo de interacciones"

2

Qué: describe el cambio

Di qué quieres añadir, quitar o modificar, en lenguaje llano.

Vago: "Mejora el contenido"

Concreto: "Sustituye el ejemplo de warfarina por un ejemplo con anticoagulantes orales directos (rivaroxabán), explicando qué tipo de interacciones son relevantes y por qué la IA no debe ser la fuente de decisión"

3

Por qué: el objetivo formativo

Si explicas el propósito, Claude Code puede adaptar mejor el tono y la estructura.

Sin contexto: "Añade una advertencia"

Con contexto: "Añade una advertencia que recuerde que los ACODs tienen un rango terapéutico diferente a los AVK y que cualquier ajuste de dosis es siempre responsabilidad del médico o farmacéutico clínico, no de la IA"

4

Cómo: el formato si tienes preferencia

Puedes indicar si quieres una lista, una alerta visual, un ejemplo de prompt, etc. Si no lo dices, Claude Code elige el formato más coherente con el resto del módulo.

Ejemplo: "Muéstralo como una alerta roja (alert-danger), igual que las otras advertencias del módulo"

Comparativa: petición pobre vs. petición efectiva

Petición pobre Petición efectiva
"Actualiza el módulo de fundamentos" "En M1.1 (¿Qué es la IA?), añade un párrafo explicando qué significa 'corte temporal' con un ejemplo farmacéutico concreto"
"Arregla la sección de recursos" "En la página de herramientas (recursos/herramientas.html), añade Elsevier ClinicalKey a la lista de bases de datos recomendadas, con una frase breve explicando para qué sirve"
"Crea un módulo nuevo" "Crea un módulo nuevo sobre IA agéntica para farmacéuticos. Debe incluir: qué es, diferencia entre chatbot/workflow/agente, 3 ejemplos en farmacia sin uso clínico, y una sección de riesgos. Tono formativo, igual que el resto de módulos"
"Cambia la portada" "Actualiza la portada para reflejar que es la versión 2 de marzo 2026. Añade una sección de novedades con los módulos nuevos. No modifiques la sección de perfiles ni los recursos adicionales"

Errores frecuentes (y cómo evitarlos)

1

Pedir demasiado de una vez

Problema: "Crea cinco módulos nuevos, actualiza todos los perfiles, corrige el glosario y cambia los colores de la web"

Por qué falla: Las peticiones grandes aumentan la probabilidad de que algún detalle se interprete mal, y si hay que corregir algo es más difícil identificar qué salió mal.

Solución: Divide en peticiones independientes. Termina y revisa una antes de pedir la siguiente.

2

No revisar el resultado antes de aprobarlo

Problema: Aceptar los cambios sin leer el contenido generado.

Por qué falla: Claude Code puede interpretar algo de forma diferente a lo que tenías en mente, o incluir un ejemplo que no se ajusta al contexto español. La IA no tiene criterio farmacéutico: tú sí.

Solución: Lee siempre el módulo o sección modificada completa antes de dar por cerrado el cambio. Sigue el mismo principio que con la dispensación: quien entrega es responsable de lo que entrega.

3

Usar términos técnicos de programación sin necesidad

Problema: "Añade un div con clase alert-info dentro del section de fundamentos con un p que diga..."

Por qué falla: No es incorrecto, pero no es necesario. Puede hacer la petición más confusa y llevar a malinterpretaciones.

Solución: Habla de contenido, no de código. "Añade una caja de información azul (como las que ya hay en el módulo) explicando que..."

4

No decir explícitamente qué no debe cambiar

Problema: "Actualiza el módulo M3" sin aclarar qué partes quieres conservar.

Por qué falla: Claude Code puede reorganizar o reformular secciones que estaban bien, creyendo que estás pidiendo una revisión general.

Solución: Sé explícito: "Solo modifica la sección X. No cambies el resto del módulo."

5

Dar el cambio por hecho si la herramienta no confirma

Problema: Asumir que el cambio se ha guardado si no aparece confirmación explícita.

Por qué falla: Si la sesión se interrumpe o hay un error, el cambio puede no haberse aplicado o guardado correctamente.

Solución: Pide siempre confirmación: "¿Se ha guardado el cambio? ¿Se ha subido al repositorio?" Y verifica abriendo la página en el navegador.

⚠️

Error de confianza excesiva en el contenido generado

Claude Code puede redactar contenido farmacéutico que suena muy correcto pero que contiene imprecisiones. La herramienta no reemplaza la revisión experta. Si un módulo habla de dosis, interacciones o protocolos clínicos, valida siempre con las fichas técnicas oficiales o guías actualizadas.

Ejemplo práctico paso a paso

Vamos a seguir un caso real completo: añadir un nuevo ejemplo de uso incorrecto de IA al módulo M2 (Casos Prohibidos), porque hemos detectado que falta un escenario sobre el uso de IA para identificar medicamentos por fotografía.

→

Situación de partida: Estás revisando el módulo M2 (modulos/casos-uso/prohibidos.html) y ves que no hay ningún ejemplo sobre identificar medicamentos por imagen. Es un error frecuente en farmacia comunitaria y quieres incluirlo.

Paso 1 — Abre la herramienta y sitúate en el proyecto

Accedes a Claude Code en el contexto del proyecto IA-farmacia-formacion. No necesitas abrir ningún archivo tú mismo; Claude Code los conoce.

Paso 2 — Formula la petición

Escribe en lenguaje natural, con los cuatro ingredientes que hemos visto (dónde, qué, por qué, cómo):

En el módulo M2, en la página de usos prohibidos (modulos/casos-uso/prohibidos.html),
añade un nuevo ejemplo de uso incorrecto: "Identificar un medicamento por fotografía".

El ejemplo debe explicar:
- Qué hace el usuario: fotografía un comprimido desconocido y le pregunta a la IA qué medicamento es
- Por qué es peligroso: la IA no puede identificar medicamentos con fiabilidad por imagen,
  puede confundir fármacos similares visualmente, y una identificación incorrecta puede
  llevar a una intoxicación o error de medicación
- Qué debe hacer en su lugar: acudir al farmacéutico, consultar la ficha técnica,
  o llamar al centro de información de medicamentos

Usa el mismo formato visual que los otros ejemplos prohibidos del módulo.
No modifiques ningún otro ejemplo existente.

Paso 3 — Claude Code lee el archivo y actúa

Claude Code leerá el archivo actual para ver el estilo y estructura existentes, y luego hará el cambio. Te mostrará un resumen de lo que ha modificado.

Paso 4 — Tú revisas el resultado

Abre el módulo M2 en el navegador y lee la nueva sección. Comprueba:

  • ¿El ejemplo está en el lugar correcto, después de los ya existentes?
  • ¿El tono es coherente con el resto del módulo?
  • ¿La información es farmacológicamente correcta?
  • ¿El resto del módulo no se ha tocado?

Paso 5 — Corriges si hace falta

Si algo no está bien, lo describes con precisión:

El ejemplo está bien pero la alternativa que propones menciona "una app de identificación de
medicamentos" como recurso. Elimina esa mención: no queremos recomendar apps no validadas.
Sustitúyela únicamente por "consultar la ficha técnica oficial en la web de la AEMPS".

Paso 6 — Confirmas que se ha guardado y subido

Claude Code habrá guardado el archivo y subido los cambios al repositorio. Puedes pedirle que confirme:

Confirma que el cambio se ha guardado correctamente y que está subido al repositorio.
✓

Resultado: El módulo M2 tiene un nuevo ejemplo bien integrado, revisado por ti como profesional, y disponible para los usuarios de la plataforma. Todo el proceso sin escribir una sola línea de código.

Resumen: reglas de oro para usar Claude Code

"Tú eres el experto farmacéutico. Claude Code es la herramienta técnica. Los dos juntos."

Claude Code puede construir y modificar la plataforma; tú garantizas que el contenido es correcto, pertinente y seguro para su uso formativo.

  • Sé específico: cuanto más contexto das, mejor resultado obtienes
  • Una cosa a la vez: peticiones pequeñas y revisadas son más fiables que cambios masivos
  • Revisa siempre: ningún cambio es definitivo hasta que tú lo validas
  • Di lo que no debe cambiar: explícitamente, si quieres preservar algo
  • Valida el contenido sanitario: la herramienta no tiene criterio clínico, tú sí
  • Pide confirmación: asegúrate de que el cambio se ha guardado y subido
